Parcourir le web

Le but du premier travail est d'écrire un programme qui parcourt des pages web en traversant des hyperliens tout en :

  • Sauvegardant localement les pages explorées
  • Collectant les adresses courriels trouvées dans les pages

Gestion des arguments

Arguments en ligne de commandes

Ton application doit prendre 3 arguments en ligne de commande dans l'ordre spécifié ici.

Attention! Il ne s'agit pas d'une application intéractive. Tu dois passer les paramètres en ligne de commandes. Il est fortement recommandé de créer plusieurs configurations d'exécution dans Intellij IDEA pour tester plusieurs combinaisons de paramètres corrects ou incorrects.

1 point Dans le cas où l'utilisateur ne fournit pas le bon nombre d'arguments tu dois lui indiquer clairement comment utiliser l'application.

  • combien d'arguments
  • quel est le rôle de chacun
  • un exemple d'utilisation
Les arguments sont:
  1. 2 points la profondeur d'exploration qui doit être un nombre entier positif.
  2. 2 points l'url de départ qui est la première page à explorer. Il doit s'agir d'une adresse valide (le format est correct et il y a une page qui y correspond).
  3. 2 points le répertoire où écrire les copies locales des fichiers explorés. Le dossier doit être accessible et on doit pouvoir y écrire.

Pour chaque argument, tu obtiens un point s'il est bien pris en compte et un point si tu affiches un message significatif quand la valeur n'est pas bonne ou absente.

Traitement

Gestion de la profondeur

3 points Le paramètre de profondeur d'exploration fonctionne ainsi:

  • Une profondeur de 0 signifie qu'on explore uniquement la page de départ fournie en argument
  • Une profondeur de 1 signifie qu'on explore la page initiale, puis les liens contenus dans la page initiale
  • Une profondeur de 2 signifie qu'on explore la page initiale, puis les liens de cette page, puis les liens contenus dans les pages liées
  • etc.

Mémoire de l'exploration

2 points Tu dois implanter un mécanisme pour ne pas visiter plus d'une fois la même page. Si une page A contient un lien vers B qui contient un lien vers A, on ne devrait pas re-explorer la page A.

Tu peux tester le bon fonctionnement en lançant ton programme avec une très grande profondeur d'exploration (ex. 1000).

Messages pendant l'exploration

1 point Tu dois fournir des messages significatifs dans la console pour chaque exploration d'une page web contenant l'URL complète du fichier.

1 point Dans le cas où une URL ne peut être explorée, tu dois fournir un message significatif incluant l'URL et le problème

Sauvegarde des fichiers explorés

1 point Chaque page html explorée doit être stockée dans un fichier avec le même nom que la resource explorée (index.html, 1.html etc.).

1 point Chaque apparition d'une adresse courriel dans le contenu d'une page doit être remplacé par votre adresse courriel avant l'écriture sur le disque.

2 points Les liens relatifs entre pages téléchargées doivent être fonctionnels et doivent pointer vers les fichiers locaux si 2 pages sont dans le même dossier.

2 points Les liens relatifs doivent fonctionner entre resources locales dans des dossiers différents.

Collecte des courriels

1 point Tu dois collecter les adresses courriels contenues dans chaque page pendant l'exploration.

1 point À la fin de l'exécution, tu dois afficher dans la console la liste des courriels collectés sans doublon dans l'ordre alphabétique.
